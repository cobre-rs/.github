# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Cobre, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email: **rogerio.alves@cobre.dev**

Please include:

- Description of the vulnerability
- Steps to reproduce
- Affected crate(s) and version(s)
- Potential impact

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Assessment:** within 1 week
- **Fix or mitigation:** depends on severity

## Scope

This policy applies to all repositories under the `cobre-rs` organization:

- `cobre` (main workspace)
- `ferrompi` (MPI bindings)
- `cobre-docs` (documentation)

## Supported Versions

Only the latest release of each crate is supported with security updates.
